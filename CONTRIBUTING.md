# Contributing

Thanks for taking the time to contribute. Here's how to get started.

## Reporting Issues

Before opening an issue, search existing ones to avoid duplicates.

When filing a bug:
- Describe what happened vs. what you expected
- Include steps to reproduce
- Add relevant logs, screenshots, or code snippets

## Submitting a PR

1. Fork the repo and create a branch from `main`
2. Name your branch descriptively: `feat/`, `fix/`, `docs/`, `chore/`
3. Make your changes — keep commits focused and atomic
4. Add or update tests if applicable
5. Ensure CI passes before requesting review
6. Open a PR against `main` and fill out the template

## Code Style

- Follow the existing conventions in the codebase
- Prefer clarity over cleverness
- Keep functions small and single-purpose
- Write comments for non-obvious logic, not obvious ones

## Commit Messages

Use the [Conventional Commits](https://www.conventionalcommits.org/) format:

```
feat: add user authentication
fix: correct timezone offset bug
docs: update API reference
chore: bump dependencies
```

## Review Process

PRs are typically reviewed within a few days. Be responsive to feedback.
Small, focused PRs get merged faster.

## Questions?

Open an issue with the `question` label or start a Discussion.
